02 System Architecture

anvil is a Bun + TypeScript CLI. It reads user input, resolves templates per language, renders files into standard project locations, and writes a lockfile for future updates.

09 Quality Toolchain: 3 Tiers

All tools unified behind make targets. Tier 1 on every commit. Tier 2 in CI. Tier 3 on demand.

Tier 1 — Every Commit

Pre-commit Hook (<30s)

  • make lint — all 21 rules
  • make format — Prettier / gofmt / Ruff
  • make typecheck — tsc / go vet / mypy
  • gitleaks — secrets on staged files
Tier 2 — CI Pipeline

make check (<5min)

  • All Tier 1 checks
  • make coverage — 80% line, 70% branch
  • make deadcode — 0 unused exports
  • make crap — complexity × coverage
  • make audit — dep vulnerability scan
Tier 3 — On Demand

make quality

  • All Tier 2 checks
  • make mutate — 70% kill rate
  • Full CRAP report
  • Too slow for CI; run before releases

10 The Anti-Slop Feedback Loop

The core insight: coding agents respond to deterministic signals. By placing lint rules in their feedback loop, agents self-correct.

11 CLI Commands

Command

anvil init

Scaffold a new or existing project.

  • --lang golang | typescript | python
  • --ci github | azure | both | none
  • Detects existing code + pkg manager
  • Provisions all tools per language
  • Runs post-scaffold install
Command

anvil update

Upgrade to newer anvil version.

  • --force allow major version bumps
  • --dry-run preview changes
  • 3-way merge via lockfile context
  • Auto-applies if user didn't edit
  • Prompts on conflict with diff
Command

anvil doctor

Verify and fix toolchain.

  • Checks runtime versions
  • Verifies lint plugin wired
  • Validates CI config present
  • Checks gitleaks, pre-commit, uv
  • Auto-fixes non-destructive only

12 AGENTS.md & Seed Code

AGENTS.md — Under 40 Lines

Complements Lint (Not Duplicates)

  • Run make check before every commit
  • Test behavior, not implementation
  • One concept per file
  • Propagate errors, don't swallow
  • Delete dead code instead of commenting
  • Descriptive names, no abbreviations
  • Prefer stdlib; consider industry-standard OSS
Seed Code — Greeter Module

Teaches By Example

  • greeter.{ext} — logic + structured logging
  • types.{ext} — exported types
  • errors.{ext} — custom error types
  • constants.{ext} — module constants
  • enums.{ext} — enum definitions
  • greeter.test.{ext} — happy + error paths
  • Every file <100 lines, every function <50
