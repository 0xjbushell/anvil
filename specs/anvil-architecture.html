02 Internal Architecture

anvil is a Bun + TypeScript CLI built with Commander, Inquirer, EJS, and Chalk. The scaffold engine resolves templates per language and renders them into standard project locations.

03 CLI Commands

Command

anvil init

Scaffold a new or existing project with anti-slop lint rules, quality tooling, CI, and seed code.

  • --lang golang | typescript | python
  • --ci github | azure | both | none
  • Detects existing project structure
  • Detects pkg manager from lockfile
  • Provisions tools per language
Command

anvil update

Upgrade scaffolded files to a newer anvil version using 3-way merge for safe, lossless updates.

  • --force allow major version bumps
  • 3-way merge: base (lockfile re-render) vs theirs (new) vs ours (disk)
  • Auto-applies if user hasn't edited
  • Prompts on conflict with diff
  • Checks new files against disk
Command

anvil doctor

Verify that all required tools are installed and configured correctly. Auto-fixes safe issues.

  • Checks runtime versions
  • Verifies lint plugin wired
  • Validates CI config present
  • Checks gitleaks, pre-commit, uv
  • Auto-fixes: non-destructive only

05 Quality Toolchain

Two tiers of quality gates. Tier 1 runs on every commit (pre-commit hook). Tier 2 runs in CI and on demand via make check. All tools are language-specific, project-local, and reproducible.

Tier 1 — Every Commit

Fast Feedback

Runs in the pre-commit hook. Subsecond to low-seconds.

  • Lint — anti-slop + structural + test rules
  • Type check — tsc / go vet+staticcheck / mypy
  • Format — Prettier / gofmt / Ruff format
  • Secrets — gitleaks pre-commit hook
Tier 2 — CI & On Demand

Deep Analysis

Full quality suite via make check. Minutes.

  • Coverage — line 80%, branch 70% (Go: line only)
  • Mutation — 70% kill rate threshold
  • Dead code — zero unused exports
  • CRAP — complexity × coverage metric
  • Security — vulnerability scanning
  • Dep audit — known vulnerability check

When anvil ships new rules or tool updates, anvil update uses 3-way merge via the lockfile context block to safely apply changes without clobbering user edits.

08 AGENTS.md — Agent Instructions

Under 40 lines. Complements the lint rules with conventions that can't be enforced by a linter. Seeded into every scaffolded project.

What It Contains

Seeded Conventions

  • Validate before commit: make check
  • Testing rules: test behavior, not implementation
  • File organization: one concept per file
  • Error handling: propagate, don't swallow
  • No dead code: delete instead of commenting out
  • Naming: descriptive, no abbreviations
  • Prefer stdlib over new dependencies
What It Doesn't

Left to Lint Rules

These are enforced mechanically by the 21 custom rules and quality gates — no need for prose instructions:

  • File/function length limits
  • Empty catch block detection
  • Console.log / print cleanup
  • TODO comment enforcement
  • Test quality requirements
  • Coverage and mutation thresholds
  • Security scanning

10 The Anti-Slop Feedback Loop

The core insight: coding agents respond to deterministic signals. By placing lint rules and quality gates in their feedback loop, agents self-correct. No human review needed for mechanical issues.

Why This Works for AI Agents

Unlike human code review (subjective, slow, inconsistent), lint rules provide:

  • Determinism — same input always produces same output
  • Immediacy — errors appear in milliseconds, not hours
  • Specificity — exact file, line, and fix suggestion
  • Completeness — every file checked every time

Agents naturally iterate when they see compiler-style errors. By encoding anti-slop heuristics as lint rules, we turn subjective "code smell" judgments into objective, fixable violations that agents handle automatically.
